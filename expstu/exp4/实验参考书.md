## 前置准备：初始化SparkSession

在开始任何操作之前，你需要先创建 `SparkSession` 对象。它是 PySpark DataFrame API 的统一入口。

**关键API提示：**

- 使用 `SparkSession.builder` 构建会话实例
- 通过 `.appName("你的应用名")` 设置应用名称
- 通过 `.master("local[*]")` 设置为本地模式运行
- 最后调用 `.getOrCreate()` 获取或创建会话

```python
from pyspark.sql import SparkSession

spark = SparkSession.builder \
    .appName("Experiment4") \
    .master("local[*]") \
    .getOrCreate()
```

> 💡 整个实验过程中只需创建一次 `SparkSession`，后续所有操作共用这个实例。

## 实验一、通过多种数据源创建DataFrame

**对应目标**：掌握通过多种数据源（包括JSON文件、Python可迭代对象、RDD以及本地文本文件等）创建DataFrame的方法。

### 1.1 从JSON文件创建DataFrame

本项目提供了一个 `students.json` 文件，包含5名学生的基本信息（id、name、age、major、gpa）。

**关键API提示：**

- 使用 `spark.read.option("multiline", "true").json("students.json")` 读取多行JSON文件（本文件为JSON数组格式，需启用multiline选项）
- 或者使用 `spark.read.format("json").option("multiline", "true").load("students.json")` 完整写法
- 读取后调用 `.show()` 查看数据内容
- 调用 `.printSchema()` 查看DataFrame的Schema结构

**操作任务：**

1. 读取 `students.json` 文件并创建DataFrame，赋值给变量 `df_json`
2. 使用 `show()` 查看数据（注意观察数据类型推断结果）
3. 使用 `printSchema()` 打印Schema，记录每个字段的推断类型

> 🔍 **思考**：JSON文件中 `gpa` 的值是浮点数，PySpark会将其推断为什么类型？`id` 呢？

### 1.2 从Python可迭代对象创建DataFrame

除了读取外部文件，你还可以直接从Python内置的数据结构创建DataFrame。

**关键API提示：**

- 准备一个Python列表，列表中每个元素是一个元组或字典
- 使用 `spark.createDataFrame(data, schema)` 创建DataFrame
- `schema` 参数可以是一个字符串列表，如 `["col1", "col2", ...]`
- 也可以传入 `pyspark.sql.types.StructType` 对象来精确控制类型

**操作任务：**

   - `("大数据技术", 4, "王教授")`
   - `("机器学习", 3, "李教授")`
   - `("数据挖掘", 3, "张教授")`
2. 使用 `spark.createDataFrame()` 创建DataFrame，指定schema为 `["course", "credit", "teacher"]`
3. 查看 `show()` 和 `printSchema()` 结果

> 🔍 **思考**：不指定schema时，PySpark推断出的字段名是什么？与手动指定有什么区别？

### 1.3 从RDD创建DataFrame

RDD（弹性分布式数据集）是Spark的底层抽象，DataFrame建立在RDD之上。你可以将RDD转换为DataFrame。

**关键API提示：**

- 先通过 `spark.sparkContext.parallelize(data)` 将Python列表转为RDD
- 然后对RDD调用 `.toDF(schema)` 方法将其转换为DataFrame
- 或者直接对RDD使用 `spark.createDataFrame(rdd, schema)` 方式转换

**操作任务：**

1. 准备一组城市数据，如 `[("北京", 2154), ("上海", 2487), ("广州", 1868), ("深圳", 1756)]` （城市名、人口/万人）
2. 使用 `spark.sparkContext.parallelize()` 创建RDD
3. 调用 `.toDF(["city", "population"])` 转换为DataFrame
4. 使用 `show()` 查看结果

> 🔍 **思考**：`.toDF()` 和 `spark.createDataFrame()` 两种方式有什么异同？

### 1.4 从本地文本文件创建DataFrame

实验文件夹中提供了 `scores.txt` 文件，每行格式为 `学号,姓名,平时成绩,考试成绩`，没有表头。

**关键API提示：**

- 需要使用 `pyspark.sql.functions.split()` 对 `value` 列按逗号分割
- 使用 `pyspark.sql.functions.col()` 引用列
- 使用 `.select()` 选取或转换列，例如 `split(col("value"), ",")` 产生一个数组列
- 通过数组下标 `.getItem(0)`、`.getItem(1)` 等提取各字段，或者直接使用 `[0]`、`[1]`

**操作任务：**

1. 使用 `spark.read.text("scores.txt")` 读取文件

## 实验二、DataFrame与RDD的相互转换

**对应目标**：掌握DataFrame与底层RDD之间的相互转换机制，并能够正确访问和解析转换后的Row对象。

### 2.1 DataFrame转RDD

**关键API提示：**

- 对DataFrame调用 `.rdd` 属性即可获得对应的RDD
- RDD中每个元素是一个 `pyspark.sql.Row` 对象
- `Row` 对象支持通过字段名访问：`row["字段名"]` 或 `row.字段名`
- `Row` 对象也支持通过位置索引访问：`row[0]`、`row[1]`（注意：PySpark默认按字段名字母序排列，请先用 `.columns` 确认顺序）
- 可以使用 `type()` 函数验证Row的类型

**操作任务：**

1. 使用实验一中创建的 `df_json`（students.json），调用 `.rdd` 转换为RDD
2. 使用 `.first()` 获取第一条数据，观察其类型
3. 分别通过字段名（如 `row["name"]`）和索引（如 `row[1]`）访问姓名字段，验证结果一致
4. 使用 `.take(3)` 获取前3条记录并遍历打印
5. 使用 `.collect()` 获取全部数据

> 🔍 **思考**：`.first()`、`.take(n)`、`.collect()` 三者的区别是什么？在数据量大时应该谨慎使用哪一个？

### 2.2 RDD转回DataFrame

**关键API提示：**

- 对RDD调用 `.toDF()` 可转回DataFrame
- 如果想保留原有schema，可以在转RDD之前先提取schema：`df.schema`
- 使用 `spark.createDataFrame(rdd, schema)` 可将RDD转为指定schema的DataFrame

**操作任务：**

1. 从 `df_json.rdd` 获取RDD
2. 使用 `.toDF()` 将RDD转回DataFrame，赋值给 `df_restored`
3. 对比 `df_restored` 和原始 `df_json` 的schema是否一致
4. 尝试使用 `spark.createDataFrame(rdd, df_json.schema)` 方式转换，对比结果

**关键API提示：**

### 7.2 使用exceptAll计算差集

**关键API提示：**

- `df_old.exceptAll(df_new)` 返回在 `df_old` 中存在但在 `df_new` 中不存在的行
- **重要区别**：`exceptAll` 保留重复行。如果某行在 `df_old` 中出现2次而在 `df_new` 中出现1次，则结果中该行保留1次

**操作任务：**

3. 反向操作：执行 `df_new.exceptAll(df_old)`，查看结果——这些是"新记录中新增"的行
4. 对比两种运算的结果，理解 `exceptAll` 的语义

> 🔍 **思考**：注意 `new_records.csv` 中 `R003` 出现了两次。在 `df_old.exceptAll(df_new)` 中，`R003` 被完全消耗了吗？为什么？

## 实验八、多表关联与分组聚合

**对应目标**：掌握通过join方法实现多表关联拼接（如内连接），并结合groupBy与agg方法完成多维度分组聚合计算。

使用 `sales.csv`（销售流水）和 `products.csv`（产品信息）两个文件。

### 8.1 内连接（Inner Join）

**关键API提示：**

- 使用 `.join(other_df, on="关联列", how="inner")` 进行表关联
- `how` 参数支持：`"inner"`（内连接）、`"left"`（左连接）、`"right"`（右连接）、`"outer"`（全连接）
- 如果两表的关联列名不同，使用条件表达式：`.join(other_df, col("id1") == col("id2"), how="inner")`
- 可使用 `.select()` 选取需要的列，避免出现重复的关联列

**操作任务：**

1. 读取 `sales.csv` 和 `products.csv`
2. 以 `product_id` 为关联键，对两表进行内连接
3. 选取 `sale_id`、`product_name`、`category`、`quantity`、`unit_price`、`region` 列
4. 查看关联结果，确认每个销售记录都匹配到了产品名称和类别

### 8.2 分组聚合计算

**关键API提示：**

- 使用 `.groupBy("列名")` 按指定列分组
- 分组后调用 `.agg(聚合函数)` 进行聚合计算
- 常用聚合函数（需从 `pyspark.sql.functions` 导入）：
  - `sum(col("列名"))`：求和
  - `avg(col("列名"))`：求平均值
  - `max(col("列名"))`：最大值
  - `min(col("列名"))`：最小值
  - `count(col("列名"))`：计数
- 可以使用 `.alias("别名")` 为聚合结果列命名

**操作任务：**

1. 在关联后的DataFrame上，先新增一列 `total_sales = quantity * unit_price`
2. 按 `region`（区域）分组，计算每个区域的：
   - 销售总额：`sum("total_sales").alias("region_total")`
   - 订单数量：`count("sale_id").alias("order_count")`
   - 平均订单金额：`avg("total_sales").alias("avg_order")`
3. 查看聚合结果

### 8.3 多维度分组聚合

**操作任务：**

1. 按 `region` 和 `category` 两个维度分组
2. 计算每组的销售总额和订单数量
3. 查看结果，分析哪个区域哪个品类销售最好

## 实验九、UDF用户自定义函数

**对应目标**：理解并掌握UDF（用户自定义函数）的注册机制，能够将其应用于DataFrame的条件过滤与数据处理中，实现复杂的Python业务逻辑。

继续使用 `employees.csv` 的DataFrame。

### 9.1 注册UDF

**关键API提示：**

- 定义一个普通的Python函数，例如 `def level(salary): ...`
- 使用 `pyspark.sql.functions.udf(python_function, return_type)` 注册为UDF
- 常用返回类型（从 `pyspark.sql.types` 导入）：`StringType()`、`IntegerType()`、`DoubleType()`
- 注册后即可像内置函数一样在 `withColumn`、`select`、`where` 中使用

**操作任务：**

1. 读取 `employees.csv`
2. 定义一个Python函数 `salary_level`，根据薪资返回等级：
   - 薪资 >= 17000 → "高级"
   - 薪资 >= 13000 → "中级"
   - 薪资 < 13000 → "初级"
3. 使用 `udf()` 注册该函数，返回类型为 `StringType()`
4. 使用 `.withColumn("level", udf函数(col("salary")))` 新增等级列
5. 查看结果

### 9.2 在where子句中使用UDF

**关键API提示：**

- UDF可以在 `.where()` 或 `.filter()` 中用于条件过滤
- 例如：`.where(udf_function(col("salary")) == "高级")`
- 注意：在 `where` 中使用UDF可能影响性能，因为Spark无法对UDF内部逻辑进行优化

**操作任务：**

1. 使用上面注册的UDF，筛选出所有"高级"员工
2. 只显示姓名、部门、薪资三列
3. 查看筛选结果

> 🔍 **思考**：UDF虽然灵活，但为什么说它可能影响Spark的执行效率？在生产环境中有什么替代方案？（提示：可以了解Spark SQL的内置函数覆盖范围）

### 10.2 withColumnRenamed 重命名列

**关键API提示：**

- `.withColumnRenamed("旧列名", "新列名")` 将列重命名
- 只能一次重命名一个列，多个列需要链式调用

**操作任务：**

1. 将 `name` 列重命名为 `employee_name`
2. 将 `department` 列重命名为 `dept`
3. 打印 `.columns` 确认重命名结果

### 10.3 selectExpr 使用SQL表达式

**关键API提示：**

- `.selectExpr("SQL表达式1", "SQL表达式2", ...)` 允许直接使用SQL表达式进行列计算
- 例如 `.selectExpr("salary * 12 AS annual", "name AS employee_name")`
- 熟悉SQL的用户会发现这种方式非常直观

**操作任务：**

1. 使用 `selectExpr` 计算年薪，并命名为 `yearly_income`
2. 同时选择 `name` 和 `department` 列
3. 查看结果

### 10.4 describe 获取描述性统计

**关键API提示：**

- `.describe()` 计算数值列的统计摘要（count、mean、stddev、min、max）
- `.describe("列名1", "列名2")` 可以指定列
- 对于字符串列，describe会返回 count、mean（空）、stddev（空）、min、max（按字典序）

**操作任务：**

1. 对 `employees` DataFrame 调用 `.describe()`，查看所有数值列的统计信息
2. 只对 `salary` 列调用 `.describe("salary")`，查看薪资的统计摘要
3. 观察结果中的 count（数量）、mean（均值）、stddev（标准差）、min（最小值）、max（最大值）

### 10.5 summary 获取更详细的统计摘要

**关键API提示：**

- `.summary("统计量1", "统计量2", ...)` 支持更多统计量
- 可用统计量包括：`"count"`, `"mean"`, `"stddev"`, `"min"`, `"25%"`, `"50%"`, `"75%"`, `"max"`
- 不带参数时，`summary()` 默认返回与 `describe()` 相同的统计量
- `summary` 比 `describe` 更强大，可以计算四分位数

**操作任务：**

1. 对 `salary` 列调用 `.summary("count", "mean", "stddev", "min", "25%",", "50%", "75%", "max")`，查看薪资的详细分布
2. 分析结果：薪资的中位数（50%）是多少？与均值（mean）相比有何差异？
